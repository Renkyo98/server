# 알려진 문제 (해결 현황)

## ✅ 해결됨 — item.c 필드 정렬 버그 (코드 버그, 수정 완료)

**증상**: 부팅 시 `[ITEM data Error] tbl=95:itemcolor` 등 다수의 아이템 파싱 오류.

**원인**: `item.c`의 아이템 읽기 루프에서, 공격/방어/순발력/HP/MP/행운/매력/회피/독/마비/수면/석화/취함/혼란/크리티컬
(총 15개) "랜덤값" 필드를 처리할 때, `readpos`(현재 읽는 필드 위치)를 추가로 1칸 증가시키는 코드가
`if( strlen(token) != 0 )` 안에 있었음. 즉 **해당 필드가 비어있는 아이템(장비가 아닌 소모품/재화/이벤트
아이템 등)은 이 추가 증가가 안 일어나서, 그 뒤 모든 필드 위치가 하나씩 밀리는 버그**였음.

**수정**: 이 증가 로직을 `if(strlen(token)!=0)` 블록 밖으로 빼서, 필드가 비어있어도 항상 실행되도록 수정.
(`item/item.c` 약 1236~1262번째 줄)

**추가로 발견해서 같이 고친 데이터 문제**:
- `itemset6.txt` 1~2번째 줄: 이름도 없이 반토막난 고장 데이터 → 삭제
- `itemset6.txt`의 21개 줄: 표준(95개) 대비 필드가 1개씩 부족 → 트레일링 콤마 추가로 보정
- `itemset6.txt`의 구간 제목 줄 3개(#없이 방치): `#` 주석 처리
